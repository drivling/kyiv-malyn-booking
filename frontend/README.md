# Frontend - Бронювання маршрутки

React + TypeScript + Vite проект для системи бронювання маршрутки Київ ↔ Малин.

## Технології

- **React 18** - UI бібліотека
- **TypeScript** - типізація
- **Vite** - збірщик та dev сервер
- **React Router** - маршрутизація

## Структура проекту

```
src/
├── api/           # API клієнт для роботи з бекендом
├── components/    # Переіспользувані UI компоненти
│   ├── Alert/
│   ├── Button/
│   ├── Input/
│   └── Select/
├── pages/         # Сторінки додатку
│   ├── BookingPage/   # Сторінка бронювання
│   └── AdminPage/     # Адмін панель
├── types/         # TypeScript типи
├── utils/         # Утиліти та константи
├── App.tsx        # Головний компонент
└── main.tsx       # Точка входу
```

## Встановлення

```bash
npm install
```

## Запуск

```bash
# Розробка
npm run dev

# Збірка для продакшену
npm run build
```

## Налаштування

API URL налаштовується через змінну оточення `VITE_API_URL` або використовується значення за замовчуванням `http://localhost:3000`.

Створіть файл `.env`:
```
VITE_API_URL=http://localhost:3000
```

## Розробка UI/UX

Проект структурований для зручної роботи з UI/UX розробниками:

- **Компоненти** винесені в окремі папки з власними стилями
- **Типи** визначені в `src/types/index.ts`
- **Стилі** модульні, легко замінювані
- **Структура** логічна та зрозуміла

### Додавання нових компонентів

1. Створіть папку в `src/components/ComponentName/`
2. Додайте `ComponentName.tsx`, `ComponentName.css`, `index.ts`
3. Експортуйте через `index.ts`
4. Імпортуйте в потрібних місцях

### Зміна стилів

Всі стилі знаходяться в окремих `.css` файлах поруч з компонентами. Можна легко замінити на:
- CSS Modules
- Styled Components
- Tailwind CSS
- Інші рішення

## API

Всі API виклики виконуються через `apiClient` з `src/api/client.ts`.

Основні методи:
- `getSchedules()` - отримати графіки
- `getBookings()` - отримати бронювання
- `createBooking()` - створити бронювання
- `checkAvailability()` - перевірити доступність місць
- І інші...
