# Час між зупинками (сегменти маршрутів)

Дані зберігаються в **`frontend/src/pages/LocalTransportPage/segmentDurations.json`**: тривалість переїзду між сусідніми зупинками в **секундах**. Ключ — `"routeId|stopFrom|stopTo"`.

## Як оновлювати після змін

### 1. Перерахувати тільки один маршрут (наприклад, 11)

Якщо ви змінили зупинки або порядок лише для одного маршруту:

```bash
node scripts/calculate_segment_durations.js --route=11
```

Інші маршрути в `segmentDurations.json` не змінюються; оновляються лише сегменти маршруту 11.

### 2. Перерахувати всі перевірені маршрути

Після змін у `frontend/public/data/malyn_transport.json` (додали/прибрали зупинки, змінили порядок) або після оновлення координат у `frontend/public/data/stops_coords.json`:

```bash
node scripts/calculate_segment_durations.js
```

Скрипт перераховує **тільки перевірені** маршрути: **2, 3, 5, 7, 8, 9, 11, 12** (список у скрипті `VERIFIED_ROUTE_IDS`).

### 3. Додати новий перевірений маршрут

Якщо ви верифікували новий маршрут (наприклад, №2 або №8) і хочете, щоб для нього теж рахувався час за координатами:

1. Відкрийте **`scripts/calculate_segment_durations.js`**.
2. Додайте ID маршруту в масив **`VERIFIED_ROUTE_IDS`**:
   ```js
   const VERIFIED_ROUTE_IDS = ['2', '3', '5', '7', '8', '9', '11'];
   ```
3. Переконайтесь, що для зупинок цього маршруту є координати в **`frontend/public/data/stops_coords.json`**.
4. Запустіть розрахунок тільки для цього маршруту:
   ```bash
   node scripts/calculate_segment_durations.js --route=2
   ```
5. У **`frontend/src/pages/LocalTransportPage/RouteMap.tsx`** і **`routeTiming.ts`** перелік перевірених маршрутів уже використовується для мапи та часу — переконайтесь, що там теж додано новий ID (наприклад, у `VERIFIED_ROUTE_IDS` у `routeTiming.ts`).

### 4. Змінити зупинки або порядок у маршруті

- Оновіть **`frontend/public/data/malyn_transport.json`** (розділ `supplement.stops.stops_by_route["номер"]`).
- Запустіть:
  ```bash
  node scripts/calculate_segment_durations.js --route=НОМЕР
  ```
- Якщо потрібно лише оновити список ключів сегментів без перерахунку часу (усі по 120 с), можна використати:
  ```bash
  node scripts/generate_segment_durations.js
  ```
  Він перезаписує **усі** сегменти перевірених маршрутів на 120 с. Після цього для реального часу знову запустіть `calculate_segment_durations.js`.

## Як працює розрахунок (calculate_segment_durations.js)

- **Відстань:** haversine за координатами з `stops_coords.json` (простіший варіант; можна згодом замінити на відстань по дорозі).
- **Час сегменту** = 12 с на зупинку (зупинитися, рушити) + час руху (відстань / швидкість).
- **Швидкість:** 35 км/год на коротких перегонах, 45 км/год на перегонах > 600 м.
- Якщо сума всіх сегментів маршруту «туди» значно більша за час «від першої до останньої по прямій», скрипт трохи зменшує час на зупинках (не більше 5–10%), щоб загальний час був ближчий до реального.

Редагувати готові значення можна вручну в **`segmentDurations.json`** (числа в секундах).
