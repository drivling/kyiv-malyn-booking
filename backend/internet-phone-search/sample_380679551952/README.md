# Зразок даних для номера 380679551952

Ця папка містить сирі дані, щоб **навчити / покращити** парсер пошуку за номером.

## Файли

| Файл | Опис |
|------|------|
| **olx.html** | HTML сторінки пошуку OLX за запитом `380679551952`: https://www.olx.ua/uk/list/q-380679551952/ |
| **result.json** | Вихід скрипта `run_internet_phone_search.py 380679551952`: посилання для пошуку, `hasData`, список витягнутих оголошень (title, url) з OLX. |

AUTO.RIA та DOM.RIA при завантаженні могли повернути помилку або іншу розмітку — їхні HTML тут відсутні. Можна повторити завантаження вручну або підставити свої збережені сторінки.

## Як використовувати для доопрацювання парсера

1. Відкрийте **olx.html** у браузері або редакторі й подивіться структуру: де в HTML знаходяться посилання на оголошення (`/d/uk/obyavlenie/...`), заголовки, блоки карток. OLX часто рендерить контент через React; частина даних може бути в `<script type="application/ld+json">` або в атрибутах `data-*`.
2. Порівняйте з **result.json**: які оголошення скрипт уже витягує, чи потрапляють лише релевантні (де в оголошенні фігурує номер), чи багато зайвого.
3. У **parse_internet_phone_search.py** у функції `parse_html_file` для `source == "olx"` можна:
   - змінити селектори (шукати по `data-cy`, класах, структурі блоків);
   - додати фільтр: залишати лише ті посилання, на сторінці яких у тексті є шуканий номер (якщо це можна визначити з HTML пошукової сторінки);
   - обмежити кількість результатів або джерела (наприклад, тільки оголошення з явним телефоном у снипеті).

Після змін перезапустіть:

```bash
cd backend/internet-phone-search
python3 run_internet_phone_search.py 380679551952
```

і порівняйте новий вивід з `result.json`.
