# Viber Оголошення - Інструкція

## Що додано

Додано можливість публікації альтернативних поїздок з Viber чатів на сайті бронювання. Ці оголошення показуються окремо від звичайних маршруток, і при спробі бронювання показується номер телефону для дзвінка.

## Запуск

### 1. Застосувати міграцію бази даних

```bash
cd backend
npx prisma migrate dev --name add_viber_listings
```

Якщо база даних не запущена, спочатку запустіть PostgreSQL:

```bash
# Для Docker
docker-compose up -d postgres

# Або якщо PostgreSQL встановлений локально
brew services start postgresql
```

### 2. Згенерувати Prisma Client

```bash
cd backend
npx prisma generate
```

### 3. Перезапустити backend

```bash
cd backend
npm run dev
```

## Використання

### Адмін панель

1. Увійдіть в адмін панель: `http://localhost:5173/admin`
2. Перейдіть на вкладку **"📱 Viber Оголошення"**
3. Натисніть **"➕ Додати оголошення"**

### Додавання оголошень

#### Одне повідомлення:

Скопіюйте повідомлення з Viber чату в такому форматі:

```
[ 9 лютого 2026 р. 12:55 ] ⁨Ковальчук Інна⁩: 2 пасажира
Сьогодні (9.02) 
Київ(академ)-Малин
18:00-18:30
0730392680
```

Вставте його в текстове поле та натисніть **"Створити"**.

#### Багато повідомлень одразу:

Скопіюйте весь блок повідомлень з чату:

```
[ 9 лютого 2026 р. 12:55 ] ⁨Ковальчук Інна⁩: 2 пасажира
Сьогодні (9.02) 
Київ(академ)-Малин
18:00-18:30
0730392680
[ 9 лютого 2026 р. 13:13 ] ⁨Максим Краснобокий⁩: Водій 09.02( сьогодні )
Малин - Київ 
Виїзд 15:30
0506661190
[ 9 лютого 2026 р. 14:25 ] ⁨Тетяна Леган⁩: Пасажир-3 особи. Малин-Житомир 
11.02.26(середа)
8.30-9.00
0679796476
```

Вставте все разом - система автоматично розпарсить кожне повідомлення.

### Що парсер розпізнає

Парсер автоматично витягує з повідомлень:

- **Ім'я відправника** з заголовка (між ⁨ та ⁩)
- **Тип оголошення**: "Водій" чи "Пасажир"
- **Маршрут**: Київ-Малин, Малин-Київ, Малин-Житомир, Житомир-Малин
- **Дату поїздки**: 
  - "сьогодні", "завтра"
  - Формат: DD.MM, DD.MM.YY, DD.MM.YYYY
- **Час відправлення**: 
  - Формат: HH:MM або HH:MM-HH:MM
- **Кількість місць/пасажирів**: числа перед словами "пасажир", "особи", "місця"
- **Номер телефону**: 
  - 10-значні номери: 0501234567
  - З кодом країни: +380501234567
  - З розділювачами: 050-123-45-67

### Перегляд на сайті

Коли користувачі обирають напрямок та дату на сторінці бронювання, вони побачать:

1. **Звичайні маршрутки** (з розкладом) - з можливістю бронювання
2. **Viber оголошення** - окрема секція з картками поїздок

При кліку на Viber оголошення показується модальне вікно з:
- Деталями поїздки
- **Номером телефону для дзвінка**
- Кнопкою копіювання номера
- Попередженням що це оголошення з чату

### Управління оголошеннями

В адмін панелі можна:

- **Деактивувати** окреме оголошення
- **Видалити** оголошення
- **🧹 Очистити старі** - автоматично деактивує оголошення, у яких дата «по» (поле `date`) раніше ніж зараз мінус 2 години (включно з сьогоднішніми)

### Фільтри

- **Тільки активні** - показувати лише активні оголошення
- Оголошення автоматично фільтруються по маршруту та даті на сторінці бронювання

## API Endpoints

Всі endpoints вимагають авторизації адміна (крім пошуку):

### Отримати оголошення

```bash
GET /viber-listings?active=true
```

### Пошук по маршруту та даті

```bash
GET /viber-listings/search?route=Kyiv-Malyn&date=2026-02-12
```

### Створити одне оголошення

```bash
POST /viber-listings
Content-Type: application/json
Authorization: admin-authenticated

{
  "rawMessage": "[ 9 лютого 2026 р. 12:55 ] ⁨Ім'я⁩: текст повідомлення"
}
```

### Створити багато оголошень

```bash
POST /viber-listings/bulk
Content-Type: application/json
Authorization: admin-authenticated

{
  "rawMessages": "весь текст з чату"
}
```

### Деактивувати оголошення

```bash
PATCH /viber-listings/:id/deactivate
Authorization: admin-authenticated
```

### Видалити оголошення

```bash
DELETE /viber-listings/:id
Authorization: admin-authenticated
```

### Очистити старі

Деактивує оголошення за **датою по** (дата поїздки + час): використовується поле `date` та кінець часу з `departureTime` — якщо час один (`15:00`), то він і є «по»; якщо діапазон (`14:30-16:00`), то «по» = `16:00`. Якщо часу немає — кінець дня (23:59). Деактивуються оголошення, у яких дата по < (поточний час − 3 години).

```bash
POST /viber-listings/cleanup-old
Authorization: admin-authenticated
```

## Структура бази даних

Таблиця `ViberListing`:

```prisma
model ViberListing {
  id            Int      @id @default(autoincrement())
  rawMessage    String   @db.Text
  senderName    String?
  listingType   String   // "driver" або "passenger"
  route         String
  date          DateTime
  departureTime String?
  seats         Int?
  phone         String
  notes         String?
  isActive      Boolean  @default(true)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

## Автоматизація (опціонально)

Можна налаштувати cron job для автоматичного очищення старих оголошень:

```bash
# Щодня о 00:00
0 0 * * * curl -X POST http://localhost:3000/viber-listings/cleanup-old \
  -H "Authorization: admin-authenticated"
```

## Приклади використання

### Сценарій 1: Оператор отримав повідомлення в Viber

1. Копіює повідомлення з чату
2. Відкриває адмін панель → Viber Оголошення
3. Натискає "Додати оголошення"
4. Вставляє текст
5. Натискає "Створити"
6. Оголошення одразу з'являється на сайті для користувачів

### Сценарій 2: Користувач шукає поїздку

1. Відкриває сторінку бронювання
2. Обирає напрямок "Київ → Малин"
3. Обирає дату
4. Бачить:
   - Розклад маршруток (якщо є)
   - Viber оголошення (якщо є на цю дату)
5. Клікає на Viber оголошення
6. Бачить модалку з номером телефону
7. Телефонує водію/пасажиру

## Підтримувані формати дати

- "сьогодні" → поточна дата
- "завтра" → завтрашня дата
- "09.02" → 9 лютого поточного року
- "09.02.26" → 9 лютого 2026
- "09.02.2026" → 9 лютого 2026

## Підтримувані формати часу

- "18:00" → 18:00
- "18:00-18:30" → 18:00-18:30
- "о 18:30" → 18:30

## Відомі обмеження

1. Парсер може не розпізнати повідомлення якщо:
   - Відсутній номер телефону
   - Незрозумілий формат маршруту
   
2. Рекомендується перевіряти створені оголошення після парсингу

3. Якщо повідомлення не розпарсилось автоматично, можна:
   - Відредагувати текст повідомлення
   - Додати оголошення вручну через API

## Питання та відповіді

**Q: Чи можна редагувати оголошення після створення?**  
A: Зараз можна тільки деактивувати або видалити. Редагування можна додати пізніше.

**Q: Чи показуються Viber оголошення якщо немає розкладу маршруток?**  
A: Так, вони показуються незалежно від наявності розкладу.

**Q: Чи можна додати маршрути окрім Київ-Малин?**  
A: Так, парсер підтримує:
- Київ ↔ Малин
- Малин ↔ Житомир
- Можна легко додати інші маршрути в парсері

**Q: Як часто треба очищати старі оголошення?**  
A: Рекомендується щодня автоматично, або вручну через кнопку "🧹 Очистити старі"
